== Project Overview

*candy-crash* is a driving training application. This is a Gleam + AffineScript web application designed to help trainees develop vehicle operation competence through pervasive ambient training.

== Technology Stack

* *Backend*: Gleam (mist, wisp)
* *Frontend*: AffineScript (typed-wasm)
* *Pattern*: TEA (The Elm Architecture) via `affinescript-tea`
* *Database*: ArangoDB (state), VeriSimDB (event history)
* *Environment*: Deno (for frontend tools), Gossamer (deployment orchestration)

== Project Structure

----
/backend      - Gleam server application
  /src/candy_crash - Application code
    /arango     - ArangoDB client
    /verisim    - VeriSimDB client
    /handlers   - HTTP request handlers
    /models     - Gleam data models
/frontend     - AffineScript web application
  /src          - AffineScript source code
  /public       - Static assets and index.html
/contractiles - ABI definitions for cross-language safety
/gossamer.conf.json - Orchestration configuration
----

== Development Setup

=== Prerequisites

* Gleam compiler
* Deno (for frontend tasks)
* VeriSimDB (running on port 8100)
* ArangoDB (for persistent state)

=== Getting Started

[source,bash]
----
# Set up environment variables
export ARANGO_URL=http://localhost:8529
export ARANGO_DATABASE=candy_crash
export SECRET_KEY_BASE=your-secret-key-base

# Start backend
cd backend
gleam run

# Build/run frontend
cd frontend
deno task build
----

== Coding Conventions

=== Gleam/Backend Best Practices

* Use strong types to enforce domain rules.
* Handle errors explicitly using `Result`.
* Keep handlers focused on HTTP/Request logic.
* Use `Context` to inject dependencies.

=== AffineScript/Frontend Best Practices

* Follow the `affinescript-tea` (Elm Architecture) pattern.
* Ensure all functions are pure when possible.
* Effects should be handled via commands.
* Maintain strict type safety for WASM interop.

== Database

* Persistent state is stored in ArangoDB.
* Event sourcing and historical attestation use VeriSimDB.
* VeriSimDB should be accessed via the port specified in `gossamer.conf.json` (default 8100).

== Testing

* Backend tests are located in `backend/test`.
* Use `gleam test` to run backend tests.

== Common Tasks

=== Adding a Backend Service

1. Define the service interface in `backend/src/candy_crash/services`.
2. Implement the client.
3. Add the service to `Context` in `backend/src/candy_crash/context.gleam`.

=== Creating a Frontend Component

1. Create a new `.as` file in `frontend/src/components`.
2. Define the `Model`, `Msg`, and `View` for the component.
3. Integrate into the main application loop.
